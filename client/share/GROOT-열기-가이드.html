GROOT 사이트가 안 열릴 때

이 프로젝트는 브라우저에서 HTTP 서버로 열어야 합니다. index.html을 탐색기에서 더블클릭(file://)만 하면 동작하지 않을 수 있습니다.

Windows PowerShell에서 npm 이 막힐 때

npm.ps1을 로드할 수 없다 / Execution_Policies 오류가 나면, PowerShell 실행 정책 때문에 npm이 막힌 것입니다.

  • 바로 해결: npm 대신 npm.cmd install, npm.cmd run dev 를 사용하거나, 명령 프롬프트(cmd)에서 npm install을 실행하세요.
  • 한 번만 설정(선택): 
    Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
    이후에는 일반적으로 npm이 동작합니다. 회사 정책으로 막히면 위 npm.cmd 방식만 쓰면 됩니다.

0) 크롬이 하얗게만 보일 때

  • client/index.html을 탐색기에서 더블클릭한 경우: 정상입니다. Vite 개발용 HTML은 file://로는 스크립트가 붙지 않습니다. 아래 1번처럼 npm run dev 후 나오는 http://localhost:… 주소로 열어야 합니다.
  • 최신 빌드(npm run build) 후에는 client/index.html 안에 안내 문구가 잠시 뜨도록 바뀌었습니다. 그래도 비어 있으면 F12 → Console 탭에 빨간 오류가 있는지 확인해 보세요.

1) 개발 모드 (API 포함, 권장)

루트에서 백엔드(예: 포트 4000)를 켠 뒤, 클라이언트 폴더에서: 

cd client
npm install
npm run dev

터미널에 나온 주소로 접속하세요. 비교 탭은 해시 라우터라 아래처럼 붙입니다.

http://localhost:5173/#/compare

포트가 5173이 아닐 수 있으니 터미널에 표시된 URL을 그대로 쓰면 됩니다.

2) 빌드 후 미리보기 (백엔드 없이 UI만 확인)

cd client
npm run build
npm run preview

기본은 4173 포트입니다.

비교 페이지 열기 (preview 켜진 경우)

3) HTML 파일로 공유 (단일 파일 업데이트)

빌드 시 JS·CSS가 하나의 index.html에 들어갑니다. 공용 복사본은 아래 명령으로 client/share/groot-latest.html 에 갱신됩니다. 

cd client
npm run build:share
  • 생성물: client/share/groot-latest.html
  • 받는 사람도 같은 폴더에서 정적 서버로 열어야 합니다. 예:
cd client/share
npx --yes serve . -p 8080

그 다음 브라우저에서:

http://127.0.0.1:8080/groot-latest.html#/compare

API(/api)가 없으면 로그인·저장 데이터는 동작하지 않을 수 있습니다. 그 경우 위 1번처럼 npm run dev와 백엔드를 함께 켜 주세요.

이 파일의 역할

GROOT-열기-가이드.html은 저장소에 포함되는 안내용 정적 HTML입니다. 앱 UI가 바뀌면 npm run build:share로 groot-latest.html만 다시 만들어 공유하면 됩니다.
